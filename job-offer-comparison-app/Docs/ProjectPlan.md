# Project Plan

**Author**: Team 039

## 1 Introduction

We propose a single-user job offer comparison app that allows the users to input, edit, and compare their current job against potential job offers while taking into consideration user-defined weighting criteria of different features of a job such as - salary, location, benefits, etc. The system will be developed for Android-based mobile devices and will include features for job entry, comparison settings, ranking, and offer evaluations.


## 2 Process Description

### **Phase 1: Inception (Requirement Analyses)**
- **Objective**: Based on the app requirements, create UML design and description documents
- **Activities**:
  - Analyze the requirements
  - Identify and refine - classes, attributes, operations and relationships in the design
  - Create a system design (UML class diagram) without involving specific language or framework 
  - Create a design description document that describes requirement realization and justification of part of the requirement that is left out in the UML design 
- **Entrance Criteria**: Requirements for Software Design, Individual design documents for team discussions 
- **Exit Criteria**: Accepted UML class diagram and design description document, highlighting similarities and differences between individual contributions and the final joint output.

### **Phase 2: Elaboration (System Design & Prototyping)**
- **Objective**: Develop project plan, use-case model, design document and test plan for the project.
- **Activities**:
  - Create **UML class diagrams, sequence diagrams, and component diagrams**.
  - Create UI mockups of the user interface of the system 
  - Create use case model for the app
  - Create test plan - different testing strategies and test cases, also assigning responsibilities for different testing types.
  - Identify potential technical challenges
- **Entrance Criteria**: Accepted UML class diagram and design description document
- **Exit Criteria**: Accepted Project Plan, Use Case Model, Design Architecture and Test Plan

### **Phase 3: Construction (Development & Testing)**
- **Objective**: Implement initial version of the app
- **Activities**:
  - Develop the app (release beta) with all the functionalities
  - Revise any documents if necessary
  - Implement SQL database for job storage
  - Conduct unit, integration, and UI testing and save initial test results
- **Entrance Criteria**: Completed Documents for Use Case Model, Design Architecture and Test Plan
- **Exit Criteria**: Fully functional prototype of the app reporting on results of the conducted initial tests

### **Phase 4: Transition (Deployment & Documentation)**
- **Objective**: Deploy the final version of the app
- **Activities**:
  - Finalize app development and resolve pending features or bugs
  - Perform final unit, integration, UI testing and regression testing and save results.
  - Package and release the **final APK**.
- **Entrance Criteria**: To have an initial version of the app that passes initial test cases
- **Exit Criteria**: **Deployed final version** of the app.
