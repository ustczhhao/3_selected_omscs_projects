# Text Transformation API

This repository contains an individual Java project that implements a reusable text-transformation library and API. The project focuses on refactoring command-line utility behavior into a programmatically invocable interface, with emphasis on exception handling, behavioral consistency, and test-driven validation.

## Project Overview

This project centers on the implementation of a Java class that exposes text-transformation functionality through a library/API interface rather than a command-line entry point. The implementation was designed to support programmatic invocation, structured error handling, and compatibility with externally provided tests.

A key part of the project was inferring expected behavior from the provided interface and test feedback, then translating that behavior into a robust implementation.

## Key Highlights

- Implemented a Java text-processing library based on a provided interface
- Refactored utility-style behavior into reusable API methods
- Replaced console-based error handling with exception-driven behavior
- Ensured compatibility with externally provided automated tests
- Demonstrated contract-based implementation and test-oriented development
- Built as an individual software engineering project

## Engineering Focus

This project emphasizes:

- Java API implementation
- Interface-driven development
- Exception design and error handling
- Behavioral inference from specifications and tests
- Test compatibility and validation
- Refactoring utility logic into reusable software components
