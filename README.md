# 3_selected_omscs_projects

A curated technical portfolio of selected graduate-level projects in machine learning, data engineering, and software development, completed through the Georgia Tech OMSCS program. This repository showcases hands-on experience in designing quantitative solutions, building software systems, and translating technical requirements into working implementations across research-oriented and engineering-oriented settings.

The projects span deep learning for medical imaging, retail data warehouse design, Android application development, and reusable API implementation. Collectively, they demonstrate strength in algorithmic thinking, software design, structured problem-solving, and practical implementation.

## Core Capabilities

- **Machine Learning and Quantitative Modeling:** deep learning, transformer-based architectures, model-oriented problem solving, and quantitative evaluation.
- **Data Engineering and Analytics:** schema design, SQL-based reporting, access control, audit logging, and business-oriented data workflows.
- **Software Engineering:** object-oriented programming, API design, exception handling, modular implementation, and specification-driven development.
- **Application Development:** Android-based mobile application design, UI behavior implementation, and input validation.
- **Engineering Discipline:** test-aware development, maintainable code organization, and translation of requirements into robust software behavior.

## Featured Projects

### 1. MRI Reconstruction with Swin-Unet
**Focus:** Deep Learning | Medical Imaging | Computer Vision

A transformer-based MRI reconstruction project for recovering high-quality knee MRI images from undersampled data. The project applies modern deep learning architecture design to an inverse imaging problem and emphasizes model implementation, reconstruction quality, and quantitative performance evaluation in a medical imaging context.

**What this project demonstrates:** deep learning workflow development, transformer-based modeling, image reconstruction, and research-oriented quantitative analysis.

### 2. BuzzBuy Data Warehouse
**Focus:** Data Engineering | SQL | Analytics Systems

A retail data warehouse project centered on schema design, role-based reporting, audit logging, and SQL-driven analytics workflows. The work focuses on translating business requirements into a structured warehouse architecture that supports reporting, governance, and decision support.

**What this project demonstrates:** data modeling, SQL, analytics-oriented database design, access control, and implementation of business-facing data systems.

### 3. Android Affine Cipher App
**Focus:** Android | Java | Input Validation

A Java-based Android application that encrypts alphanumeric text using an affine cipher, with structured input validation and specification-driven UI behavior. The project emphasizes correctness of user-facing logic, clean handling of edge cases, and disciplined implementation against defined behavioral requirements.

**What this project demonstrates:** Java programming, Android UI development, validation logic, and careful implementation of functional requirements.

### 4. Job Offer Comparison App
**Focus:** Mobile Development | Software Design | Decision Support

An Android application for comparing job offers across salary, benefits, and location-adjusted compensation factors. The project focuses on software design, structured comparison logic, and user-oriented application flow for multi-factor decision support.

**What this project demonstrates:** end-to-end application development, object-oriented design, application logic design, and implementation of practical decision-support functionality.

### 5. Text Transformation API
**Focus:** API Design | Java | Software Testing

A Java project implementing a reusable text-processing API with interface-driven design, exception-based error handling, and behavior aligned with formal testing requirements. The project emphasizes modularity, predictable software behavior, and maintainable API construction.

**What this project demonstrates:** API design, interface-based architecture, exception handling, modular code structure, and test-compatible implementation.
